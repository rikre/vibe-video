## 项目介绍

**VibeVideo** 是一个面向漫剧创作者、短剧制作团队和 AI 视频运营的 **Web 工作台**。它将传统短剧制作中割裂的「剧本 → 角色 → 分镜 → 配音 → 成片」流程，收敛到一个统一的工作台中，通过 Agent 自动化能力实现「黑灯工厂」式的流水线生产。

### 解决什么问题

| 痛点 | VibeVideo 方案 |
|------|---------------|
| 剧本到分镜靠人工拆解，耗时长 | Agent 自动解析剧本，拆章节、识别场景与对白 |
| 角色/场景/道具资产分散管理 | 统一资产工作台，Agent 自动提取并生成资产卡 |
| 分镜提示词编写门槛高 | Agent 根据剧本自动生成镜头提示词 |
| 配音、字幕、合成工具割裂 | 成片工作台集成播放器、模块 Tab、时间轴 |
| 创作流程不可视、进度不可控 | 统一工作台 + 集数导航 + 实时状态反馈 |

### 双创作模式

| 模式 | 适用场景 | 流程 |
|------|---------|------|
| **剧本模式** | 已有完整剧本，希望 Agent 全流程自动生产 | 上传剧本 → Agent 解析 → 资产提取 → 分镜生成 → 成片合成 |
| **自由模式** | 创意驱动，边做边调 | 手动创建资产 → 编排分镜 → 生成视频 → 成片合成 |

> 两种模式的成片页面结构、交互、功能完全一致，均使用统一工作台。

## Agent 黑灯工厂模式

VibeVideo 的核心是 **Agent 黑灯工厂**——用户只需投入「剧本」原料，Agent 流水线自动完成解析、提取、生成、合成全流程，最终产出可导出的成片。

### 工作原理

```
┌─────────────────────────────────────────────────────────────┐
│                    Agent 黑灯工厂流水线                       │
│                                                             │
│  ┌──────────┐    ┌──────────┐    ┌──────────┐    ┌────────┐ │
│  │ 剧本解析  │───▶│ 资产生成  │───▶│ 分镜生成  │───▶│成片合成│ │
│  │  Agent   │    │  Agent   │    │  Agent   │    │ Agent  │ │
│  └──────────┘    └──────────┘    └──────────┘    └────────┘ │
│       │               │               │               │      │
│       ▼               ▼               ▼               ▼      │
│   章节结构         角色卡          分镜表          成片预览    │
│   场景列表         场景卡          镜头提示词       导出文件    │
│   对白片段         道具卡                         (视频+音频   │
│                   音色建议                        +字幕+BGM)  │
└─────────────────────────────────────────────────────────────┘
```

### 四大 Agent 职责

#### 1. 剧本解析 Agent

| 输入 | 输出 |
|------|------|
| 剧本文件（.txt / .docx） | 章节结构、场景列表、对白片段 |

- 自动读取剧本，拆分章节与场景
- 识别角色对白与旁白
- 提取场景描述与情绪标签

#### 2. 资产生成 Agent

| 输入 | 输出 |
|------|------|
| 章节结构、场景列表 | 角色卡、场景卡、道具卡、音色建议 |

- 从剧本中提取角色、场景、道具
- 为每个角色生成视觉设定卡
- 根据角色特征推荐匹配音色

#### 3. 分镜生成 Agent

| 输入 | 输出 |
|------|------|
| 资产卡、章节结构 | 分镜表、镜头提示词 |

- 按集数拆分镜头
- 为每个镜头生成画面描述与 AI 绘图提示词
- 标注镜头类型（特写/中景/全景）与运镜方式

#### 4. 成片合成 Agent

| 输入 | 输出 |
|------|------|
| 分镜视频、配音、字幕、BGM | 成片预览与导出文件 |

- 合并分镜视频片段
- 对齐配音音轨与字幕时间轴
- 混入背景音乐
- 输出可预览、可导出的成片

### 黑灯工厂的「无人值守」特性

| 特性 | 说明 |
|------|------|
| **原料驱动** | 用户投入剧本，Agent 自动启动全流程 |
| **阶段串联** | 前一阶段产出自动作为下一阶段输入 |
| **状态可视** | 每个阶段有明确的等待/进行中/已完成状态 |
| **异常可重试** | 单个镜头或音轨失败不影响整体流程，支持局部重试 |
| **人工介入点** | 资产确认、分镜确认环节允许用户校对后继续 |

### 自由模式的差异

自由模式跳过剧本解析，用户手动搭建资产后，从分镜生成阶段进入 Agent 流水线：

```
手动创建资产 → 分镜编排 → 视频生成 → 成片合成
                              ↑
                    Agent 从此处接入
```

## 核心功能

- **成片页**：统一工作台（集数导航 + 播放器 + 模块 Tab + 时间轴 + 导出）

### 成片统一工作台

| 区域 | 功能 |
|------|------|
| 集数导航栏 | 左侧纵向集数列表，点击切换 |
| 顶部操作栏 | 成片预览标题、导出/下载按钮 |
| 播放器卡片 | 视频预览舞台 + 播放控件 |
| 模块 Tab | 分镜视频 / 配音 / 配字幕 三模块切换 |
| 时间轴 | 时间标尺 + 分镜轨道（点击切换模块） |
| 模块面板 | 根据当前模块展示操作入口 |

### 配音弹窗

- 四维筛选器：语言 / 搜索 / 性别 / 年龄
- 实时过滤 + 空状态 + 重置筛选
- 音色卡片网格展示

### 导演工作台（3D 构图与机位调度）

导演工作台是 VibeVideo 的 **3D 场面调度工具**，对标 LibTV 导演台，用轻量级 3D 交互帮助创作者完成构图与机位设计，并产出可被 AI 图像/视频模型理解的参考图。

#### 核心功能思路

导演台的核心价值在于：**将抽象的「镜头语言」转化为可视化的 3D 操作，再转化为 AI 模型可理解的提示词**。

```
3D 构图操作 → 机位截图 → Qwen LoRA 提示词 → AI 多视角参考图 → 分镜视频
```

创作者在 3D 场景中摆放角色、调整机位、截图，系统自动将机位参数（方位角/仰角/距离）映射为 `<sks>` 提示词，驱动 AI 生成一致视角的参考图，最终串联到分镜视频生成流程。

#### 模块与子模块

| 模块 | 子模块 | 职责 |
|------|--------|------|
| **Header** | 顶部导航栏 | 品牌标识、导演/机位视角切换、云端同步状态指示、全屏 |
| **ObjectTree** | 左侧场景树 | 场景对象搜索、机位/角色/元素分组展示、多选打组、可见性控制 |
| **Stage** | 3D 视口（核心） | React Three Fiber 渲染：人体素模、几何体、GLB 模型、机位标记、全景背景球 |
| **Toolbar** | 底部工具栏 | 变换工具（移动/旋转/缩放/吸附）、添加菜单、预设机位、截图、撤销重做 |
| **Inspector** | 右侧属性面板 | 属性 Tab（对象/机位参数）、场景 Tab（环境/全景/辅助）、截图 Tab（管理/发送） |
| **CaptureDialog** | 截图对话框 | 5 种比例裁剪、离屏渲染、异步上传、机位关联 |
| **CrowdDialog** | 群众阵列 | 行列网格配置、间距调节、自动打组 |
| **ProjectSync** | 云端同步引擎 | 防抖自动保存、乐观锁冲突合并、离线重试、页面隐藏保存 |
| **CanvasShell** | 无限画布 | 导演台节点 + 截图节点 + SVG 连线 + 缩略地图 |

#### 技术亮点

- **React Three Fiber** 真实 3D 渲染（非 CSS 模拟），支持 GLB/glTF 模型导入
- **性能自适应**：>24 对象自动切换 InstancedMesh + 阴影降级
- **Zustand + persist**：localStorage 持久化 + 50 步撤销/重做栈
- **乐观锁云端同步**：If-Match revision + 409 冲突自动重试 + 离线指数退避

> 完整技术方案见 [导演工作台技术方案](docs/导演工作台技术方案.md)

## 设计系统

VibeVideo 采用 **「暗房霓虹」（Darkroom Neon）** 品牌语言，灵感来自暗房红灯与霓虹灯管的对比美学。

### 品牌色

| Token | 色值 | 用途 |
|-------|------|------|
| `--accent` | `#c0cf62` Moss Lime | 主色，按钮/激活态/强调 |
| `--accent-hover` | `#d9f43c` 霓虹莱姆 | 悬停态/渐变终点 |
| `--accent-soft` | `rgba(192,207,98,0.14)` | 激活态背景/柔和填充 |
| `--accent-glow` | `#d9f43c` | 发光效果 |

### 字体

| 字体 | 用途 |
|------|------|
| **Sora** | 标题与展示文字 |
| **Hanken Grotesk** | 正文与 UI 文字 |
| **JetBrains Mono** | 数据、代码、时间戳 |

### 主题

支持 **暗色**（默认）与 **亮色** 双主题，所有组件通过语义 Token 自适应切换。

### 可访问性

- 全局 `focus-visible` 焦点环（WCAG 2.4.7）
- `prefers-reduced-motion` 减弱动效
- ARIA tablist / tab / tabpanel 语义
- 图标按钮 `aria-label` 标注

> 完整设计规范见 [DESIGN.md](DESIGN.md)

## 快速开始

### 环境要求

- Node.js 18+
- npm 或 pnpm

### 安装与运行

```bash
# 克隆仓库
git clone https://github.com/rikre/vibe-video.git
cd vibe-video

# 安装依赖
npm install

# 启动开发服务器
npm run dev
```

开发服务器默认运行在 `http://localhost:5173`

### 构建

```bash
# 类型检查
npx tsc --noEmit

# 生产构建
npm run build
```
