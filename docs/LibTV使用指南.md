#### 脚本节点 🎬

##### 🚩脚本功能全新升级

- **功能概述：****从“基础脚本工具”升级到“分镜故事板脚本工具”。**

    - 新版脚本不只是把剧本拆成表格，而是把剧本拆解、剧本资产管理、分镜头调整、提示词合成、批量生图、批量生视频串成一条完整流程。

    - 适合流水漫、AI 短剧、动画短片、故事分镜等内容创作场景，可以更快完成：剧本拆解 → 角色/场景/道具资产提取 → 分镜脚本调整 → 批量生成分镜图 → 批量生成视频片段。



- 对比脚本旧版 解决的问题

**目前脚本工具在线上节点中渗透排名第三，仅次于图片和视频，**用户对“从文字到画面”的批量生产需求是比较明确的。

针对收集的用户问题，新版脚本方案围绕四个关键词展开：**资产化、可编辑、可批量、灵活修改**

- **资产化**：将脚本从“纯文本表格”升级为“内容 \+ 资产”结构，风格、角色、场景、道具等资产与脚本绑定并可被提示词引用；

- **可编辑**：脚本的每一行、每一个字段、每一个资产都可以被用户修改，修改后能同步更新对应的提示词；脚本中的分镜支持新增、删除、调整顺序。

- **可批量**：用户可批量执行分镜图和分镜视频的生成。可自由选择生成范围（单个/部分/全部）、生成类型（批量生图、批量生视频）、以及批量执行的模型类型。

- **灵活修改：**新版脚本可以在批量生成前直接补充和调整，用户可在真正批量生图、生视频前，把角色、场景、提示词、参考图都修改补充到比较稳的状态，再进入生产。



- 三、使用方式功能介绍

    - **第 1 步：添加脚本节点**

        - **可通过两种方式**添加脚本节点：

            - 第一种，双击画布，或者打开底部的 “\+”，选择「脚本 new」。

            - 第二种，先把剧本文本输入到一个文本节点里，再从文本节点拉出一条线，选择「脚本 new」。

        - 脚本节点的**前置输入**可以是一段**剧本文本**，也可以同时接入一些**参考图**，比如角色图、场景图、风格图等。

            - 如果只是想快速拆分镜，输入剧本文本就可以开始。

            - 如果希望后续生成的角色、画风、场景更稳定，建议在一开始就把关键参考图接进去，这样脚本节点在拆解时也可以更好地理解整体风格和角色设定。

- **第 2 步：开始生成**

将文本、图片节点与脚本节点连接后**，**或者直接在脚本节点的生成器输入框内输入剧本内容，选择合适的脚本拆解模型，**点击生成。**

- **模型的选择**：推荐优先选择CVLM5\.5 或 GVLM 3\.1 

- 生成后，脚本节点会自动完成几件事：

    - 把剧本文本拆成结构化 shot； 

    - 提取关键角色、场景、道具； 

    - 整理每个 shot 对应的画面信息； 

    - 为后续生图、生视频准备基础提示词。 

这一步相当于先把一段完整剧本拆成一份可编辑的分镜故事板。

- **第 3 步：编辑脚本节点**

**这一步是后续生成是否准确的关键。**

脚本节点生成出来后，**不建议直接批量生图/视频**。更好的方式是先检查 shot 信息，管理好角色、场景、道具等资产，再进入后面的批量生成。

因为后续所有分镜图和视频片段，都会基于这里的信息继续生成。如果这里的角色描述不准、场景不清楚、镜头动作太弱，后面批量生成就会把这些问题放大。

新版脚本不是只服务单张图或视频片段，而是更适合一整段故事的生产。

五、创作者测评

新版脚本更适合的场景是：

1. AI影视/动画短片的灵感泛化、资产拆解、分镜生成

2. 短剧漫剧的剧集单集资产拆解、素材生产


**一些局限**：

1. 无法做到精品短片一键直出，导演思维、时长的判断、镜头与镜头的衔接还是会有瑕疵。

2. 针对TVC场景的产品一致性会有不稳定的情况。

1. **新版脚本比较适合的场景主要有两类。**

    1. AI 影视 / 动画短片

        1. 适合做灵感泛化、剧本拆解、资产整理和分镜生成。

        2. 如果只是有一个故事想法，可以先让脚本节点拆成分镜，再看每个 shot 是否合理。

        3. 如果已经有比较完整的剧本，也可以直接用它来提取角色、场景、道具，快速生成一版故事板。

        4. 对早期创意验证来说，这个流程会比较高效。

    2. 短剧 / 漫剧单集素材生产

        1. 短剧和漫剧更依赖批量生产，一集里面经常有几十个镜头，而且角色和场景会反复出现。

        2. 新版脚本在这种场景下比较有优势：

            1. 可以快速拆单集分镜； 

            2. 可以统一管理角色和场景； 

            3. 可以批量生成分镜图； 

            4. 可以继续批量生成视频； 

            5. 可以先局部测试，再批量跑全量。 

        3. 对于流水线生产来说，它能减少很多重复操作。



2. **当前还存在的边界**

    1. 新版脚本目前还不是“精品短片一键直出”。

        1. 导演思维、镜头节奏、时长判断、镜头与镜头之间的衔接，还是需要人工介入。

        2. 尤其是精品 TVC 这类对产品一致性、镜头设计、品牌质感要求特别高的场景，当前生成稳定性还会有一些不确定性。

    2. 但整体来看，相比旧版，V2 的生产逻辑更完整，生成链路也更顺。对于需要快速拆剧本、搭故事板、批量出素材的创作者来说，新版脚本会比旧版更实用。

3. **克隆流程**

    **1️⃣ 朗读音频**

    建议在相对安静的环境下,进行清晰、语速适中的朗读,尽量读115s以上,这样克隆出的音色结果才更准确。如果希

    望有某些对应的情绪,可以在朗读的时候加上,这样使用该音色的的时候,也会带上对应的情绪。

    **2️⃣ 生成并试听音色**

    音色生成成功后,先用一段短文本试听。试听时重点检查:

    1. 相似度：是否接近原声音色

    2. 清晰度：是否有杂音、糊音、爆音

    3. 稳定性：是否忽高忽低、忽快忽慢

    4. 自然度：是否像真人说话

    5. 情绪表达：是否符合文本内容

    3️⃣ **填写音色相关的信息,进行音色信息登录**

    登录相关音色信息,完成后即可完成克隆音色的创建。

    4️⃣ 使用克隆音色生成语音

    音色克隆成功后,**在语音生成页面选择该音色,再输入文本生成语音。**

    注意:如果创建音色后,7天内未使用,会被minimax官方api清除\(受内存限制,有实效性\),所以建议创建完立刻

    使用一次,这样就会一直保存在自己的音色库中。
