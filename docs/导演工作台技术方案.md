# 菠萝AI · 导演工作台 — 技术方案

> 基于 LibTV 导演台功能需求，结合开源项目 Aholo Viewer 与 Qwen-Image-Edit-2511-Multiple-Angles-LoRA 设计

---

## 一、功能概述

### 1.1 产品定位
导演工作台是菠萝AI漫剧创作平台中的**3D 构图与机位调度工具**，对标 LibTV 导演台，用轻量级 3D 交互帮助创作者完成场面调度，并产出可被 AI 图像/视频模型准确理解的构图参考。

### 1.2 核心能力清单

| 能力域 | 功能点 | 当前状态 |
|--------|--------|---------|
| 场景搭建 | 角色/道具/群演/机位 添加与编辑 | ✅ 已实现 |
| 站位调度 | 移动(V)/旋转(R)/缩放(S) + 网格吸附 | ✅ 已实现 |
| 机位管理 | 机位视角切换、FOV 焦距、注视目标跟随 | ✅ 已实现 |
| 截图产出 | 多比例截图、机位列表管理、发送到画布 | ✅ 已实现 |
| 全景背景 | 全景图上传作为场景背景 | ✅ 已实现 |
| 属性编辑 | 名称/颜色/位置/旋转/缩放/姿势/形状 | ✅ 已实现 |
| 分组管理 | 打组/解组 | ✅ 已实现 |
| 机位→提示词 | 96 姿态映射 Qwen LoRA 提示词 | ✅ 已实现 |

## 三、开源项目集成方案

### 3.1 Aholo Viewer 集成（3D 渲染引擎）

**用途：** 替换 Phase 1 的 CSS 2.5D 模拟，提供真实 3D 渲染、碰撞检测、相机避障

**集成步骤：**

```bash
npm install @manycore/aholo-viewer
```

**核心 API 接入：**

```typescript
import {
  createViewer, setViewerConfig,
  PerspectiveCamera, BackgroundMode,
  Vector3, Color, SplatLoader, SplatUtils,
} from '@manycore/aholo-viewer';

// 1. 创建 3D 视口
const viewer = createViewer('director-viewport', container, {});

// 2. 透视相机（对应导演台机位）
const camera = new PerspectiveCamera(fov, aspect, 0.1, 2000);
camera.position.set(x, y, z);
camera.lookAt(new Vector3(targetX, targetY, targetZ)); // 注视目标

// 3. 加载场景资产（素模角色 / 道具 / 全景图）
const splat = await SplatUtils.createSplat(data);
viewer.getScene().add(splat);

// 4. 配置渲染管线
setViewerConfig(viewer, {
  pipeline: {
    Splatting: { enabled: true },
    Background: { background: { active: BackgroundMode.BasicBackground } },
  },
});

// 5. 截图（对应导演台"截图建机位"）
viewer.render(); // 触发渲染
const canvas = viewer.getCanvas();
const dataUrl = canvas.toDataURL('image/png'); // 获取截图
```

**碰撞检测集成（机位防穿墙）：**

```typescript
// 从 3DGS 场景生成体素碰撞体
const collider = await VoxelCollider.fromScene(splat);

// 第三人称相机避障
function updateCameraWithCollision(targetPos: Vector3) {
  const ray = new Ray(camera.position, targetPos);
  const hit = collider.rayCast(ray);
  if (hit) {
    // 相机被阻挡，调整到碰撞点前方
    camera.position.lerp(hit.point, 0.9);
  } else {
    camera.position.copy(targetPos);
  }
}
```

**性能配置（三档）：**

| 模式 | 适用场景 | 配置 |
|------|---------|------|
| 效果优先 | 桌面端预览 | TAA 开启、高精度 Splatting |
| 性能优先 | 常规使用 | TAA 关闭、标准 Splatting |
| 极限性能 | 移动端 | 降采样、简化管线 |

**映射函数（已实现）：**

```typescript
function buildCameraPrompt(azimuth: number, elevation: number, distance: number): string {
  const azimuthLabel = AZIMUTH_LABELS[azimuth];
  const elevationLabel = ELEVATION_LABELS[elevation];
  const distanceLabel = DISTANCE_LABELS[distance];
  return `<sks> ${azimuthLabel} ${elevationLabel} ${distanceLabel}`;
}
// 输出示例: <sks> front view eye-level shot medium shot
```

## 六、技术决策说明

### 6.1 为何 Phase 1 使用 CSS 2.5D 而非直接集成 Aholo？
1. **快速验证产品形态**：先用纯前端验证交互逻辑和用户流程
2. **降低集成风险**：Aholo 需要浏览器 GPU 加速和 3DGS 数据准备，先跑通 UI 再接入
3. **保持零额外依赖**：Phase 1 不增加包体积，与现有项目技术栈完全一致

### 6.2 为何选择 Aholo Viewer 而非 Three.js？
1. **碰撞检测开箱即用**：体素碰撞体 + 第三人称相机避障直接对应导演台需求
2. **3DGS 真实场景支持**：可扫描真实片场导入，这是 Three.js 不具备的
3. **性能领先**：10亿高斯点 vs Three.js 传统 Mesh，大场景优势明显
4. **MIT 协议商用友好**

### 6.3 为何选择 Qwen LoRA 而非其他视角控制方案？
1. **96 姿态精准映射**：与导演台机位参数一一对应
2. **3D 一致性**：基于高斯泼溅数据训练，跨视角一致
3. **低角度强化**：-30° 仰拍是影视常用机位，该 LoRA 特别优化
4. **开源可商用**：HuggingFace 开源，LiblibAI 已上架
