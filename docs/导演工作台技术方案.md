# 菠萝AI · 导演工作台 — 技术方案

> 基于 LibTV 导演台功能需求，结合开源项目 Aholo Viewer 与 Qwen-Image-Edit-2511-Multiple-Angles-LoRA 设计

---

## 一、功能概述

### 1.1 产品定位

导演工作台是菠萝AI漫剧创作平台中的 **3D 构图与机位调度工具**，对标 LibTV 导演台，用轻量级 3D 交互帮助创作者完成场面调度，并产出可被 AI 图像/视频模型准确理解的构图参考。

### 1.2 核心能力清单

| 能力域 | 功能点 | 当前状态 |
|--------|--------|---------|
| 场景搭建 | 角色/道具/群演/机位 添加与编辑 | ✅ 已实现 |
| 站位调度 | 移动(V)/旋转(R)/缩放(S) + 网格吸附 | ✅ 已实现 |
| 机位管理 | 机位视角切换、FOV 焦距、注视目标跟随 | ✅ 已实现 |
| 截图产出 | 多比例截图、机位列表管理、发送到画布 | ✅ 已实现 |
| 全景背景 | 全景图上传作为场景背景 | ✅ 已实现 |
| 属性编辑 | 名称/颜色/位置/旋转/缩放/姿势/形状 | ✅ 已实现 |
| 分组管理 | 打组/解组 + 组内联动变换 | ✅ 已实现 |
| 撤销重做 | 50 步历史栈 + 快捷键 ⌘Z/⇧⌘Z | ✅ 已实现 |
| 云端同步 | 项目自动保存 + 乐观锁冲突合并 | ✅ 已实现 |
| 无限画布 | 截图发送到画布节点 + 连线展示 | ✅ 已实现 |
| 本地模型 | GLB/glTF 上传导入 + 自动归一化 | ✅ 已实现 |
| 性能自适应 | >24 对象自动切换 InstancedMesh | ✅ 已实现 |

## 五、开源项目集成方案

### 5.1 Aholo Viewer 集成（3DGS 渲染引擎）

**用途：** 替换当前 R3F 素模，提供真实 3DGS 场景渲染、碰撞检测、相机避障

**核心 API 接入：**

```typescript
import {
  createViewer, setViewerConfig,
  PerspectiveCamera, BackgroundMode,
  Vector3, Color, SplatLoader, SplatUtils,
} from '@manycore/aholo-viewer';

// 1. 创建 3D 视口
const viewer = createViewer('director-viewport', container, {});

// 2. 透视相机（对应导演台机位）
const camera = new PerspectiveCamera(fov, aspect, 0.1, 2000);
camera.position.set(x, y, z);
camera.lookAt(new Vector3(targetX, targetY, targetZ));

// 3. 加载 3DGS 场景
const splat = await SplatUtils.createSplat(data);
viewer.getScene().add(splat);

// 4. 碰撞检测（机位防穿墙）
const collider = await VoxelCollider.fromScene(splat);
function updateCameraWithCollision(targetPos: Vector3) {
  const ray = new Ray(camera.position, targetPos);
  const hit = collider.rayCast(ray);
  if (hit) camera.position.lerp(hit.point, 0.9);
  else camera.position.copy(targetPos);
}
```

## 八、技术决策说明

### 8.1 为何选择 React Three Fiber 而非纯 CSS 2.5D？

1. **真实 3D 交互**：R3F 提供真实的 3D 空间操作，TransformControls 开箱即用
2. **模型导入**：直接支持 GLB/glTF 加载，CSS 方案无法实现
3. **性能可控**：InstancedMesh 支持大规模对象，DPR 自适应
4. **生态丰富**：drei 提供 OrbitControls / Grid / Html 等成熟组件
5. **迁移友好**：后续 Phase 2 迁移到 Aholo Viewer 时，相机/对象模型可复用

### 8.2 为何选择 Zustand 而非 Redux？

1. **轻量**：无样板代码，store 定义即类型安全
2. **persist 中间件**：一行代码实现 localStorage 持久化
3. **细粒度订阅**：组件按需订阅字段，避免不必要的重渲染
4. **与 R3F 兼容**：useDirectorStore 可直接在 R3F 组件内调用

### 8.3 为何选择乐观锁而非 CRDT？

1. **简单可靠**：revision + If-Match 头实现冲突检测
2. **语义清晰**：409 冲突时自动拉取最新版本并重试保存
3. **离线友好**：断网时切换 offline 状态，恢复后自动同步
4. **适用场景**：单人或低频协作场景下，乐观锁足够

### 8.4 为何选择 Aholo Viewer 而非 Three.js（Phase 2）？

1. **碰撞检测开箱即用**：体素碰撞体 + 第三人称相机避障直接对应导演台需求
2. **3DGS 真实场景支持**：可扫描真实片场导入，这是 Three.js 不具备的
3. **性能领先**：10亿高斯点 vs Three.js 传统 Mesh，大场景优势明显
4. **MIT 协议商用友好**

### 8.5 为何选择 Qwen LoRA 而非其他视角控制方案？

1. **96 姿态精准映射**：与导演台机位参数一一对应
2. **3D 一致性**：基于高斯泼溅数据训练，跨视角一致
3. **低角度强化**：-30° 仰拍是影视常用机位，该 LoRA 特别优化
4. **开源可商用**：HuggingFace 开源，LiblibAI 已上架
